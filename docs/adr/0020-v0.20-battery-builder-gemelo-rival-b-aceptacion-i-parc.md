# ADR 0020 — battery_builder + gemelo (rival b) + aceptación (i) parcial (para auditoría humana)

- **Versión**: v0.20
- **Fecha**: 2026-06-11
- **Estado**: Vigente

---

- **`battery_builder`** (ARCHITECTURE §6): candidatos sobre la superficie de control → desacuerdo entre {verdad, rivales} (capeado) × relevancia (stakes + taper de cohort) → selección **estratificada por dosis** (top-K) + cola de auditoría → `battery.json`. Determinístico/seeded.
- **Rival (b) gemelo** (`derive_rivals.rival_twin`): mecanismo con un operador ablado (vía `meta.ablation`) + refit de momentos al pool corrupto. Conectado al set de desacuerdo (falla in-support → la batería pesa donde la trampa muerde).
- **Aceptación (i) — side-by-side hand vs derivada** (ajuste 5): la batería 100% derivada **preserva el orden de la escalera** (R: 1.0 > 0.965 > 0.797 > 0.305 > 0 > 0) y el **spread naive↔canonical** (0.969 vs hand 0.955), con cobertura de dosis 0.4–9.9. **PERO el margen verdad→perturbado (0.035) queda < 5%**. Cadena de investigación (registrada, disciplina): concentración off-support por extrapolación del no-latente → taper de cohort → cap de desacuerdo → estratificación por dosis. **Hallazgo residual**: el ancla **S_ingenuo (rung_5) extrapola catastróficamente a dosis alta** en la batería derivada (pesada ahí) → **infla el denom** (2.59 vs hand 0.95) → comprime el margen de la degradación más chica. **Es la misma clase que v0.12** (ancla patológica infla el rango), ahora para S_ingenuo. *Razón: no se fuerza el pass (regla v0.19: no debilitar rivales para inflar márgenes); el bootstrap NO expira aún; queda para la auditoría humana + decisión de margen.*
