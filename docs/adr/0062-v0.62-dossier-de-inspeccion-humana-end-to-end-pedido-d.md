- **`wager/report/episode_report.py`**: un HTML autocontenido por caso, tres vistas en orden — **lo que el examinador instaló** (operadores con knobs/ablaciones, fuentes con sus corrupciones explicitadas, schema del entregable, funcional con cláusula verbatim), **lo que el agente vio** (el brief), **la certificación** (gates como chips, escalera R, techo alcanzable) y **cada episodio turno a turno**: razonamiento del solver (colapsable), celdas python, salidas, verbos con costos, submits rechazados marcados en rojo, y la submission final calificada. Tooling de reporte, jamás reward path; salida a `reports/` (gitignored, regenerable): `python -m wager.report.episode_report cases/<case_id>`. Generado para `selection_bias_v0` (4 episodios, incluida la autopsia del #19) y `dummy_dose_v0`; publicado además como Artifact para lectura inmediata.
