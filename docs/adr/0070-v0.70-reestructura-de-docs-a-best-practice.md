# ADR 0070 — Reestructura de docs a estructura best-practice

- **Versión**: v0.70
- **Fecha**: 2026-07-05
- **Estado**: Vigente

---

**Orden de Lucas** ("los docs crecieron brutalmente... un doc que dice el norte no puede tener tantas líneas"), tras investigar buenas prácticas (Diátaxis, guía de Anthropic para Claude Code, patrón ADR, context engineering). Decisión: **un documento por propósito, superficie siempre-cargada mínima, todo lo demás bajo demanda**.

- **`WIKI.md` (nueva)**: la pata *Explicación* de Diátaxis que faltaba — entender de cero, sin jerga, en profundidad, con estado honesto (✅/🔨/❓). Es lo que un humano lee para entender el proyecto. Renderizable a artifact.
- **`README.md` (nuevo)**: puerta de entrada de una pantalla (qué es, setup, correr, mapa de docs).
- **NORTH_STAR DISUELTO** (no se mantiene): su *por qué* llano → `WIKI.md`; el diseño (§3/§4) queda cubierto por `ARCHITECTURE.md`; el ethos/reglas → `CLAUDE.md`; el red-team (§7) → `docs/red-team.md`; el programa (§6) → `docs/roadmap.md`; las open-questions (§10) → `docs/open-questions.md`. El texto completo se preserva en `docs/archived/NORTH_STAR_full.md` (nada se borra; las citas "NORTH_STAR §N" resuelven ahí).
- **Decision Log → `docs/adr/`**: el blob append-only de 700 líneas se partió en 69 ADRs (uno por decisión, patrón adr.github.io) + índice. Los identificadores v0.NN se preservan en filename+body para que las citas viejas resuelvan. Esta entrada es la #70.
- **`CLAUDE.md` a punteros**: se sacó el "leer NORTH_STAR primero"; ahora es operativa lean con el mapa de docs. **`CURRENT_STATE.md`** absorbió `E1_MATRIX.md` (cartera). `MUNDOS_DINAMICOS_CONTEXT.md` archivado.
- **Guardias adaptadas** (con autotest, regla ADR 0057): `check_decision_log.py` pasó de block-diffing un archivo a inmutabilidad file-level de `docs/adr/*.md` (should-pass/should-fail verificados); los lints de version-sync de headers se retiraron (sin "versión actual" única con ADRs per-file); se mantuvo el lint de trazabilidad (brief_clause verbatim) + secciones clave, y se agregó un lint de completitud del índice de ADRs.
- **Método** (respeta ADR 0056): splits determinísticos por línea + Edits puntuales, JAMÁS regex bulk sobre docs constitucionales; cada fase con la suite verde y su commit.

*Razón: la constitución monolítica (NORTH_STAR 900+ líneas mezclando por qué + diseño + decisiones + changelog) violaba el principio de "un doc, un trabajo" y cargaba contexto pesado cada sesión. La estructura nueva separa los cuatro modos de Diátaxis y hace que un humano lea UN doc (WIKI) para entender y Claude cargue solo lo que la tarea pide.*
