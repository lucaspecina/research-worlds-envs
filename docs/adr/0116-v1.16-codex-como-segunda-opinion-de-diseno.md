# ADR 0116 — Codex (GPT-5.6 Sol) como segunda opinión de diseño/reflexión, NO como escritor de código

- **Versión**: v1.16
- **Fecha**: 2026-07-10
- **Estado**: Vigente

---

Orden de Lucas: incorporar a Codex (OpenAI) al workflow como **compañero de pensamiento y crítica**,
y dejar escrito cómo se trabaja con él.

- **Rol**: segunda opinión / crítica / pulido de ideas en cuestiones de **diseño, reflexión,
  análisis de resultados** — una tercera lectura independiente. **NO escribe código** (la solución la
  pensamos y escribimos nosotros; a Codex se le pide *pensar la solución, analizar, criticar*).
- **Cuándo**: ante una **decisión importante y no trivial**, o cuando Lucas lo pide. Claude puede
  **ofrecer** consultarlo ("¿querés que le consulte a Codex?") antes de hacerlo, no dispararlo solo
  para cosas triviales.
- **Cómo (setup, ver memoria `codex-setup`)**: CLI 0.144.1; modelo **`gpt-5.6-sol`** / esfuerzo
  **`max`** (default en `~/.codex/config.toml`). El ID correcto es `gpt-5.6-sol` — `gpt-5.6` y
  `gpt-5.6-codex` los rechaza el login de ChatGPT; `max` solo existe en los 5.6. **Sesión
  PERSISTENTE**: `codex exec --skip-git-repo-check "..."` y continuar con `codex exec resume --last
  "..."` (probado: retiene el contexto entre invocaciones). Alternativas disponibles: plugin codex
  (skills `codex:rescue`/`codex:setup`, agente `codex:codex-rescue`) y MCP (skill `codex-collab`).
- **Reporte**: lo que diga Codex se traduce a Lucas EN LLANO (memoria `comunicar-sin-jerga`).

Documentado en `CLAUDE.md` (sección "Codex — segunda opinión / pensar juntos"), que se carga en cada
sesión.

*Razón: sumar una voz crítica independiente para las decisiones no triviales (el proyecto ya valora
las auditorías cruzadas: cuando dos lecturas independientes convergen, la foto es más robusta). Se
acota a PENSAR, no a codear, para no delegar la construcción ni diluir la autoría.*
