# ADR 0072 — Consolidaciones finales de docs (auditoría doc-por-doc de Lucas)

- **Versión**: v0.72
- **Fecha**: 2026-07-05
- **Estado**: Vigente

---

Tras la reestructura (ADR 0070), Lucas auditó doc por doc preguntando "¿esto es best-practice?". Tres cierres para que no quede ningún monolito ni doc que duplique-y-drife:

- **ARCHITECTURE (287→índice) → `docs/reference/*.md`**: era un monolito de ~385 líneas; una referencia conviene descomponerla para cargar solo el tema relevante (context engineering). Split mecánico por sección (contenido verbatim, verificado): `world-model` (§1-2,10-11 incl. contrato de ventana §10.1), `operators` (§3-4), `rivals-battery` (§5-6), `scoring` (§9), `certificates` (§7,13), `harness` (§8,12,14). ARCHITECTURE.md queda como índice corto; las citas "ARCHITECTURE §N" resuelven vía ese índice. El lint de secciones se adaptó: chequea que los 6 archivos de reference existan (con su should-fail).
- **CURRENT_STATE → `docs/roadmap.md`**: duplicaba resultados (ya en ADRs) + cartera (plan) + deudas (open-questions), y era el doc más propenso a desactualizarse. Se fundió: roadmap ahora es *estado actual (volátil) + cartera E1 + programa E1→E4*. `CURRENT_STATE.md` removido; CLAUDE/README repunteados.
- **`REDTEAM.md` (raíz) → su test**: enumeraba escapes del sandbox que `tests/test_sandbox_redteam.py` ya implementa — los tests SON el checklist. Docstring del test actualizado (los tests son la lista); notas originales archivadas en `docs/archived/REDTEAM_sandbox.md`. (Distinto de `docs/red-team.md`, el registro de amenazas del proyecto, que se queda.)

*Razón: best-practice no es "menos archivos" sino "un doc, un trabajo; superficie siempre-cargada mínima; todo lo demás bajo demanda". La reestructura había disuelto NORTH_STAR y el Decision Log pero dejado ARCHITECTURE monolítico y CURRENT_STATE duplicando — Lucas lo cazó. Con esto, ningún doc activo es monolito ni duplica.*
